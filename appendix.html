Technical Appendix

An overview of the project's code structure and key dependencies.

Code Structure: A Modular Pipeline

The project is organized into two primary modules, each representing a core engine of the GridOS platform.

Physical Prediction Pipeline

A sequential, five-script pipeline for generating data and training the GNN:

  • 01_ingest_grid_topology.py: Builds the static grid graph in Neo4j.
  • 02_generate_simulation_data.py: Runs Pandapower simulations to create scenarios.
  • 03_preprocess_for_gnn.py: Queries Neo4j and creates the PyTorch Geometric data file.
  • 04_train_gnn_models.py: Trains and benchmarks GNN models (GCN vs. GAT).
  • 05_evaluate_and_visualize.py: Evaluates the model and generates output visuals.

Economic Forecasting Pipeline

A comprehensive script for data processing, model benchmarking, and reporting:

  • 01_build_feature_set.py: Ingests and merges multiple real-world NYISO CSVs.
  • run_forecasting_models.py: Trains Transformer, XGBoost, and SARIMA models, evaluates performance, and generates the final HTML report.
